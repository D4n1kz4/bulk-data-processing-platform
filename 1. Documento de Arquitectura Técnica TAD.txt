PROYECTO

1. Descripción General
Objetivo

Desarrollar una plataforma fullstack orientada al procesamiento masivo de datos utilizando:

Spring Boot
Angular
Microfrontends
PostgreSQL

El proyecto tendrá como objetivo principal demostrar conocimientos técnicos relacionados con:

arquitectura backend enterprise
microfrontends
seguridad
procesamiento masivo de datos
diseño REST API
integración frontend/backend
validaciones
auditoría
paginación
modularización frontend
buenas prácticas de desarrollo


2. Enfoque del Proyecto

El proyecto estará orientado principalmente a:

fortalecer portafolio profesional
demostrar conocimientos técnicos modernos
practicar arquitectura enterprise
construir experiencia realista fullstack
simular entornos corporativos modernos


3. Arquitectura General del Sistema

La plataforma estará dividida en:

Frontend Microfrontend Layer
            ↓
        REST API
            ↓
    Spring Boot Backend
            ↓
       PostgreSQL


4. Arquitectura Frontend

El frontend será construido utilizando Angular con arquitectura basada en microfrontends mediante Module Federation.

4.1 Enfoque Microfrontend

El sistema utilizará Module Federation para cargar aplicaciones frontend independientes.

Objetivos
modularización
desacoplamiento
separación por dominio
despliegues independientes
escalabilidad frontend

4.2 Shell Application

Aplicación principal encargada de:
navegación
layout
autenticación
carga de microfrontends
rutas principales

4.3 Microfrontend Customers

Responsable de:
listado de clientes
búsqueda
filtros
paginación
creación y edición de registros

4.4 Microfrontend Imports

Responsable de:
carga de archivos CSV
visualización de resultados
historial de importaciones
visualización de errores

4.5 Microfrontend Audit
Responsable de:

visualización de logs
auditoría de acciones
filtros de actividad


5. Arquitectura Backend

9.2 Customers

POST /api/import/customers

9.4 Audit

10. Seguridad

La plataforma implementará:

JWT Authentication
autorización basada en roles
BCrypt Password Encoding
DTO Validation
Protected Routes
CORS Configuration


11. Procesamiento Masivo de Datos

La plataforma permitirá:

importar archivos CSV
procesar miles de registros
validar datos automáticamente
detectar registros inválidos
generar reportes de errores
Validaciones

Ejemplos:

email válido
campos obligatorios
duplicados
formato inválido
estados incorrectos
